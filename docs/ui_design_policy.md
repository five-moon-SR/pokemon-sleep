# UI設計方針 — 脱・AIテンプレ構成（2026-07-23策定）

配色ではなく**画面構成・ボタン配置**の設計方針。Claude(参考サイト実地調査)とCodex Terra(コードレビュー)の合議で策定。
参考: RaenonX(文脈常設・タスクナビ) / だいふくチェッカー(1画面1目的・物理ボタン) / ゲーム本体(浅い画面×階層深掘り)。

## 診断: AI感の正体
全ページが「設定群 → 同形の情報箱 → 同形ボタン」の生成テンプレ構造になっている。
機能一覧で画面を作っており、**ユーザーの行動フロー（今この画面で何を決めるか）から設計されていない**。
- 均質カードの等間隔グリッド、全カード下に同形「詳細」ボタン
- 全セクション同じ重さの線見出し・border付きコンテナの縦積み
- selectbox/multiselectの海（だいふくは進化回数を0/1/2の物理ボタンでワンタップ）
- ユーザー前提(RR/フィールド/狙いレシピ)が画面に埋没（RaenonXはヘッダ常設チップ）

## 配置原則（全画面共通）
1. 画面最上部には「**今の前提**」か「**今の答え**」のどちらか一つだけを置く
2. **primaryボタンは画面に1つ**。状態を前進させる操作（登録する/この編成を使う）だけをprimaryに
3. 2〜6択の定型判断 → **物理ボタン（pills/segmented_control）**。検索対象（種族/レシピ/食材/サブスキル）→ selectbox
4. **popover** = その場の主目的を遮らない設定（詳細フィルタ・週次条件・保存名）
5. **dialog** = 詳細閲覧・内訳・破壊操作の確認（主画面の縦長化を防ぐ）
6. **expander** = 参考情報のみ。「今決めるのに必須の項目」を隠す用途に使わない
7. 枠線コンテナは主アクション・異常・比較対象だけ。説明セクションを枠で囲わない

## 画面別の再設計

### 所持ポケデータ（owned）→ 目的「探す」
- 最上部 = 検索欄 + よく使う絞り込みpills（得意/ランク帯/現在vs将来性）。12個のselect群は `popover("詳細フィルタ")` へ
- 選択中の条件はチップで常設 + 「すべて解除」
- 表示方式(カード/表)でなく**タスクで切る**: 探す=1列コンパクト結果行 / 比較する=選択2〜5体だけ表 / 全列テーブルは詳細側へ退避
- 詳細dialogは「個体を理解する」だけに。評価再計算(保存副作用)と削除は分離

### パーティー編成（party）→ 目的「今週の5体を決める」
- 最上部 = 文脈ストリップ（例: シアン / 好み: オレン・モモン / 食材寄せ / 2倍週）。編集は `popover("今週の条件")`
- 料理カテゴリ・役割プリセット・イベントは物理ボタン化。役割5スライダーはカスタム時のみ
- 主画面を `おまかせ / 手動で組む` の segmented に分割
  - おまかせ: 上位3案＋「なぜこの案か」＋この編成を使う
  - 手動: 「次に埋めるべき役割」を1つ提案し候補5件（全役割タブ廃止）
- **編成済み5枠は候補より上**（文脈ストリップ直下）。分析(食材内訳/余剰/つなぎ)は `dialog("この編成の内訳")` へ。保存は5枠直下の popover

### 食材・育成戦略（ingredients）→ 目的「今週の料理の穴を埋める」
- 最初に料理1〜3件 → 直下にhero「不足食材 / 料理可能回数 / 次の一手」
- `料理の穴 / 育成する / 捕まえる / 食材索引` のsegmentedでモード分割。初期=料理の穴（狙いレシピの食材だけ表示）
- 不足食材ごとに次アクションを1つ（編成で試す→party導線 / 育成対象を見る）
- 19食材マトリクスは「食材索引」モードに隔離

### 個体登録（register）→ 2段階化
- 種族選択後: ①必須（Lv/性格/現在Lvの食材枠）→ ②任意（将来食材/サブスキル/リボン/メモ、開くまで隠す）
- Lvは数値入力＋ゲーム節目のクイックボタン（1/10/25/30/50/60）
- 性格は上昇軸pills→selectbox（直接検索も残す）
- 登録成功後に「続けて登録」「この個体を開く」

## 実装ロードマップ（各段デプロイ可能）
1. **共通土台**: app.pyナビのグループ化（今週の運用/個体管理/データ・ガイド）+ `context_strip`/`result_row`/`decision_card` を components に追加
2. **owned検索体験**（最優先・最小リスク）: 簡易フィルタ+詳細popover分割、結果行中心化、詳細dialogの副作用分離
3. **register段階化**: 必須/任意分割+Lv節目ボタン
4. **party再編**: 文脈ストリップ+おまかせ/手動、分析dialog退避、保存位置移動
5. **ingredientsモード分解**: 初期表示=料理の穴

検証基準: 375px幅で「**最初の10秒で次に押す場所が分かるか**」。
