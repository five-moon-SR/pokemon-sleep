# 2026-05-08 個体UI整合化＋だいふく依存撤去（夕方セッション）

## 何があったか
1. 個体登録（`register.py`）を「**だいふくチェッカー結果のコピペ必須**」から「**全手入力**」に再設計。
2. 個体強化（`update.py`）／登録修正（`edit_record.py`）／所持ポケ（`owned.py`）を新しい register に揃える形で整合化。
3. 性格選択を「軸カテゴリ → 性格名」の2段階セレクトに統一し、下降軸を併記する `format_nature_label()` を共通ヘルパに集約。

## 実装・決定の中身

### `views/register.py`：全手入力フォームに再設計
- **撤去**：だいふくチェッカーへのリンク、コピペ欄、`parse_daifuku_paste()`、`CIRCLED_DIGITS`、`_truncate_pct`、`DAIFUKU_CHECKER_URL` の import、`math` / `re` の import。
- **入力フィールド**（ステップ1〜4）：
  1. 種族選択（変更なし、ひらがな検索対応）
  2. 基本情報：Lv（1-65 / -10 / +10 ボタン）／性格カテゴリ＋性格／メインスキルLv（1-8）
  3. 食材スロット2/3（radio）
  4. サブスキル5枠 Lv10/25/50/75/100（selectbox、金→青→白→名前順）
  5. 任意：ニックネーム／🎀 おやすみリボン／メモ
- **先行入力可ポリシー**：食材スロット2/3 とサブスキル5枠は Lv に関わらず常時表示。ゲーム上で抽選候補・解放予定が事前に見えるため、「Lv未到達でも記入できる」ことを優先。バリデーションだけ Lv 連動（Lv≥30 ならスロット2必須／Lv≥60 ならスロット3必須）。
- **登録時の保存**：`level / caught_level / current_level` は手入力Lvに統一（旧 `level=だいふく評価Lv 60固定` の運用は新規登録では発生しなくなる）。`daifuku_rank / daifuku_eval_type / daifuku_eval_percent` は新規登録で NULL（DB列は残置）。

### 性格選択の2段階セレクト＋共通ヘルパ
- `constants.py` に新設：
  - `NATURE_AXIS_GROUPS: list[tuple[str, list[str]]]` — グループ表示順 = 無補正 / おてつだいスピード ↑ / げんき回復 ↑ / 食材確率 ↑ / メインスキル発生 ↑ / EXP獲得 ↑
  - `NATURE_DOWN_AXIS_LABEL: dict[str, str]` — `speed→おてスピ↓` `energy_recovery→げんき↓` `ingredient→食材↓` `skill→スキル↓` `exp→EXP↓`
  - `find_nature_axis(name)` — 性格名から所属軸グループ名を逆引き
  - `format_nature_label(name)` — 「ようき（食材↓）」「がんばりや（無補正）」「—」（None時）形式の統一表示
- `register.py` / `edit_record.py` 双方で共通利用。register は新規入力、edit_record は既存値から軸を自動判定して初期表示。
- 軸カテゴリを変更したら下段の性格セレクトを `on_change` でリセット（不整合防止）。

### `views/update.py`：候補一覧を所持ポケと同形に
- 候補一覧テーブルの **ランク／評価%／評価タイプ** を `daifuku_*` 由来から **自前評価 v1.2**（`evaluate_at_levels`）に差し替え。
- **Lv50ランク／Lv50%／Lv60ランク／Lv60%／全体ランク／全体%** を追加（owned.py と同じ列順）。
- 性格列は `format_nature_label` で軸併記表示。
- サブスキル編集の値ラベル「未解放」→「**未入力**」（先行入力可になったため、`NULL = 未到達` の意味づけが破綻したため）。

### `views/edit_record.py`：登録修正ページを整合
- 性格 `text_input`（自由入力）→ `register.py` と同じ **2段階セレクト**（軸＋性格名）に。
- **🎀 おやすみリボン編集**を新規追加（段階セレクト＋アイコンプレビュー、register と同じ仕様）。
- **だいふく評価セクション（rank / pct / eval_type 編集UI）を撤去**。DB列・既存値・送信時保存ロジックも削除（差分メッセージの該当行も削除）。
- サブスキルのプレースホルダ「（未解放）」→「（未入力）」に。
- 不要になった import を整理（`DAIFUKU_RANK_EMOJI`, `DAIFUKU_RANKS`, ローカルの `_rank_label` 削除）。

### `views/owned.py`：表示・フィルタの整合
- フィルタラベル「**だいふくランク**」→「**ランク（自前評価・種族内）**」（実態は自前ランクに効いていたため）。
- 性格列を `format_nature_label` で軸併記表示に統一。
- VIEW_PRESETS「🍡 だいふく（旧）」は **既存個体の参考表示として温存**。
- 評価器の `eval_type_source='daifuku'` 経路（既存個体の登録値からラベル化）も温存。

### サブスキルのセマンティクス変更（影響範囲広い）
- `subskill_lvN = NULL` の意味が「**未解放**（=ゲーム上未到達）」から「**未入力**（=ユーザー未記入）」に変化。
- Lv 未到達の表示装飾（灰色イタリック、アイコン非表示など）は owned.py の Styler でビュー側のみ実施。データ層は意味を持たない。

## 残課題・未確定事項
- **だいふく列の最終撤去**：DB列・owned.py 旧ビュー・evaluator の `eval_type_source='daifuku'` 経路はまだ残存。すべての既存個体で自前評価の精度が安定したら撤去候補。
- **`DAIFUKU_RANKS` 文字列セットの一般化**：`["増田","SS","S","A","B","C","D"]` を owned.py のランクフィルタが流用しているが、定数名が「だいふく」のまま。改名するか `RANKS` のような中立名に作り直すかは未決。
- **`level` カラムの扱い**：旧 `level=だいふく評価Lv60固定`、新 `level=手入力Lv`。DB上に新旧の混在が生じている。当面は混在のまま（自前評価・個体強化はいずれも `current_level` を見るので実害は薄い）。整理するなら `level` カラムを `eval_level` あたりに改名、または廃止する判断が要る。
- **既存個体の性格データ**：旧 text_input 由来で NATURES 集合外の値が入っている可能性あり。edit_record.py 側では「軸=未指定 + options 先頭に元値」でフォールバック表示するが、ユーザーが明示的に再選択しないと正しい25種に揃わない。
- **`game_pokemon_sleep_play_context.md`** の「UI未」表記が outdated（朝のセッションで home.py に編集ブロック追加済み）。次回タッチアップ。
