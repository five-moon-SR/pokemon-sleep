# 2026-05-08 食材獲得期待値の校正＋qty解釈バグ修正

## 何があったか

`utils/food_expectation.py` を v0.1 → v0.2 へ。だいふく期待値チェッカー（pokemonsleepdaifuku.com/checker/expected/）と校正することで、**誤差 0.4%** で計算ロジックを合致させた。同時に、評価器・party.py 側に潜んでいた **食材qty解釈バグ** を発見・修正。

## v0.2 の主な変更点

### 1. 1日のおてつだい時間モデル
- 旧 v0.1: `active_hours × 3600`（PlayContext.active_hours、平日16.5h前提）
- 新 v0.2: `DAILY_EFFECTIVE_ASSIST_SECONDS = 132,888秒` 固定（だいふく互換）
- 根拠：ポケスリ Wiki「げんき/おてつだい時間」より、げんき値帯ごとの時間倍率（150〜81=×0.45 / 80〜61=×0.52 / …）を 24h × 60 = 1440分の推移シミュで積分すると 2214.80分 = 132,888秒。
- 計算式：`1日のおてつだい回数 = 132,888 / 個体のおてつだい時間(秒)`

### 2. データ化
- 新規 `data/genki.json`：げんき区間テーブル＋日合計実効秒数の定数を保存
- 新規 `utils/genki.py`：`DAILY_EFFECTIVE_ASSIST_SECONDS` 公開定数＋ `get_time_multiplier(genki_value)` ヘルパ

### 3. 食材qty 解釈の根本修正（**重大バグ修正**）
- **誤った旧解釈**：`master.ingredients.X.qty` リストは「Lv1/Lv30/Lv60 の段階値」
- **正しい解釈**：「**スロット位置順** での qty」
  - a枠食材（qty list 長さ3）= 第一/第二/第三スロットでの qty
  - b枠食材（qty list 長さ2）= 第二/第三スロットでの qty（第一では取れない）
  - c枠食材（qty list 長さ1）= 第三スロットでの qty
- 例：リザードンマメミート(a枠) `[2, 5, 7]` → 第一スロットで2、第二で5、第三で7
- Lv は **枠の解放（b=Lv30, c=Lv60）にだけ効き、qty には効かない**

### 4. 影響を受けたコード

| `views/party.py` | `_ingredient_qty_for_level()` で同様の誤り | `qty_at_slot()` を使用 |

`utils.food_expectation.qty_at_slot(species, food_name, slot_idx)` と `find_food_origin(species, food_name)` を public 化して共有ヘルパに昇格。

## 追記（2026-05-08 後半）：28ケースキャリブ完了 → v0.3 補正式確定

### キャリブ結果
- 25/30 行が ±10% 以内、平均誤差 4.5%、最大誤差 17.0%
- baseline / サブスキル単独 / 性格補正 / 補正積乗 / ABC編成 / c枠なし種族 / 好物OFF：誤差±1〜3% 以内
- 不一致ケースは v0.2 未対応の3種類の補正に集中

### v0.3 で実装すべき式（全て確定）

#### A. team-buff（おてつだいボーナス持ちチームメンバー）
- 線形 +5% / 人
- `total_count = 自身装着(0/1) + チーム他メンバーの装着数`
- `speed_mult ×= 1.0 + 0.05 × total_count`
- 検証: case 021〜024 で daifuku と±0.4% 整合

#### B. 「げんき常に80」オプション
- 食材獲得 × 1.179（種族非依存）
- 検証: リザードン×1.179 / プクリン×1.194

#### C. 自身げんき回復ループ（げんきオール/エール/チャージ系）
- `self_recovery_per_day = (1日のスキル発動回数 × スキル効果値) / 5`
- `food_mult ×= 1.0 + 0.00374 × self_recovery_per_day`
- 「げんき常に80」ON のときは無効化（げんき上限到達でループしない）
- 検証:
  - プクリン case 029→009: 自身回復 20.25/日 → +7.58%（係数で +7.56%、一致）
  - サーナイト推定: 自身回復 57.2/日 → +21.4%（case 016 の誤差 -17%≒+20.5% と整合）

### 副次的な発見
- daifuku は **24h 稼働** 前提（睡眠時間入力なし）。`DAILY_EFFECTIVE_ASSIST_SECONDS = 132,888秒` が正解
- 食材獲得は **「好きなきのみ」ON/OFF の影響を受けない**（きのみエナジーは ×2/×1 だが食材は同値）
- ジュカインの「きのみバースト」max_lv：data/main_skill.json では 6 だが daifuku では 7 まで入力可（**要データ修正候補**）
- case 005 のスキル発動回数が daifuku で「オン」と誤表示されていた（**要再確認**）
