# ChatGPT 共有用ファイル置き場

ポケスリ案件で ChatGPT を「思考・設計支援」担当として使うための、共有ファイルの置き場。
齋藤さんが ChatGPT セッションに手動でアップロードする想定。Claude Code（実装担当）はこのフォルダの内容を最新に保つ。

## フォルダ構成

```
docs/chatgpt/
├ README.md                    ← 本ファイル
├ project_instructions.md      ← ChatGPTのプロジェクト「指示欄」に貼る本文
├ 00_core_briefing.md          ← 大本のオンボーディング（最初に必ず渡す）
└ updates/                     ← 日々の進捗ファイル（追加していく）
   └ YYYY-MM-DD_<topic>.md
```

## 渡し方の流れ

### 初回セットアップ（ChatGPTのプロジェクトを作るとき、1回だけ）
1. ChatGPT で「プロジェクト」を新規作成（例：「ポケスリ管理ツール」）
2. `project_instructions.md` の囲み内本文を、プロジェクトの「指示」欄に貼る → 全会話で常駐するシステムプロンプトになる
3. プロジェクトの「ファイル」に `00_core_briefing.md` をアップロード

### 新規 ChatGPT セッションを始めるとき（プロジェクト内で）
1. 上記セットアップ済みなら、コア・ブリーフィングと指示は自動で読まれる
2. その時点で関連が深い `updates/` のファイルがあれば一緒に渡す
3. 質問する

### 既存セッションを継続するとき
- 直近の `updates/` だけ追加で渡す（コア・ブリーフィングは既に読まれている前提）

## ファイル種別

### `00_core_briefing.md`（コア・ブリーフィング）
プロジェクト全体の常駐コンテキスト。約12KB / A4 6〜7ページ。
**大きな機能追加・設計変更があったら更新する**（特に「9. ロードマップ」「8. 評価チェッカー」は変動が早い）。

### `updates/YYYY-MM-DD_<topic>.md`（日々の進捗ファイル）
1日分の作業内容や、ある程度まとまった機能追加・設計変更の差分。
コア・ブリーフィングを毎回書き換えるとデカくなりすぎるので、**新しい知見・実装・決定はここに加点的に蓄積**する。

ファイル名: `YYYY-MM-DD_<英語スネークケースの短い説明>.md`
例: `2026-05-08_home_extension.md` / `2026-05-09_party_food_expectation.md`

中身のテンプレ：
```markdown
# YYYY-MM-DD <topic>

## 何があったか
（1〜3行で）

## 実装・決定の中身
- 具体的に何を加えた／変えたか
- 影響を受けるファイル
- 数式や構造

## 残課題・未確定事項
- ...

## ChatGPTに今後相談したいこと（あれば）
- ...
```

### 質問パケット（必要なときだけ）
ChatGPT に何かを聞きたいときは、`00_core_briefing.md` §13 のテンプレに沿って自己完結したパケットを作る。Claude Code に「ChatGPTに聞く文章作って」と言えば、そのパケットを生成する。

質問パケットは原則ファイル化しない（一度きりの会話入力なので）。回答が来て採用された設計は `updates/` に追記する形で残す。
