# きのみエナジー計算式（AI読み取り用ノート）

## 目的
`utils/berry_energy.py` に実装した計算ロジックの根拠と検証データをまとめる。
**JSON化はしない方針**（`game_pokemon_sleep_calc_policy`）。式があれば全Lvを再現できる。

## ソース
- `貼り付けデータ集/きのみエナジー詳細データ.txt`（PukiWiki形式、Wiki由来）
- `data/berry.json` の `base_energy`（Lv1の獲得エナジー＝基礎値）

## 中核の式

### 1個あたりの獲得エナジー（ポケモンLv別）
```
lv_energy(base, lv) = round( max( base + (lv-1), base * 1.025^(lv-1) ) )
```
- レベルが低い時は線形項 `base + (lv-1)` が支配
- Lv20前後から指数項 `base * 1.025^(lv-1)` が支配
- 小数点以下は四捨五入

### カビゴンへの最終エナジー
```
表示エナジー = ceil( lv_energy × (1 + field_bonus) × (1 + favorite_bonus) )
実際エナジー = ceil( lv_energy × (1 + field_bonus) ) × (1 + favorite_bonus)
```
- favorite_bonus = 1（=2倍）または 0
- 表示と実際で括弧の取り方が異なるため、「奇数ズレ」が生じる演出仕様
- サーバー同期（睡眠計測終了時など）で表示は実際値に修正される
