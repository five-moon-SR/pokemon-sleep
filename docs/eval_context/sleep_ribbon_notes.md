# おやすみリボン仕様（AI読み取り用ノート）

## ソース
- `貼り付けデータ集/おやすみリボン.txt`（Wiki Ver.1.10.0実装）

## 概要
チームに編成して睡眠計測することで蓄積する「**一緒に眠った時間**」が一定量に達するごとに、おてつだいポケモンに自動付与されるリボン。
所持数ボーナス＋（進化残り回数に応じた）おてつだい時間短縮 が得られる。
**リボンを装着しなくても効果は発揮される**（外見への反映のみが装着の意味）。

## 4段階×進化残り回数のテーブル

| 段階 | 達成時間 | 所持数+(累計) | 進化残0時間倍率 | 進化残1時間倍率 | 進化残2時間倍率 |
|---:|---:|---:|---:|---:|---:|
| 1 | 200h | +1 | 1.00 | 1.00 | 1.00 |
| 2 | 500h | +3 | 1.00 | **0.95** | **0.89** |
| 3 | 1,000h | +6 | 1.00 | 0.95 | 0.89 |
| 4 | 2,000h | +8 | 1.00 | **0.88** | **0.75** |

短縮率(%)で表現すると:
- 段階2: 残1=-5% / 残2=-11%
- 段階4: 残1=-7%(累計-12%) / 残2=-14%(累計-25%)

段階3は所持数+3 ＋ プロフィールアイコン獲得のみ（時間短縮の追加なし）。

## 進化残り回数の判定ルール
- 0 = 最終進化形（時間短縮なし、所持数ボーナスのみ）
- 1 = あと1回進化可能（例: ピカチュウ）
- 2 = あと2回進化可能（例: ピチュー）

ポケスリ実装済の進化先のみカウント。例: マンキー→オコリザル→コノヨザル(未実装) はマンキーが「残1」扱い。

特殊フォームで最終進化扱いになるもの:
- ピカチュウ(ハロウィン) — ライチュウへ進化不可
- ピカチュウ(ホリデー) — 同上

## 時間短縮の合成（重要）
おやすみリボンの時間短縮は **性格・サブスキルとは別軸で乗算**。

```
最終おてつだい時間 = 基準おてつだい時間
  × ribbon_multiplier(species, stage)
  × nature_speed_factor   # 性格(▲おてスピなら×0.9, ▼なら×1.075)
  × subskill_speed_factor # 合算後（35%上限）の倍率
```

例: ピチュー（進化残2）が以下を持つ場合
- リボン4 → ×0.75
- おてつだいスピードM → ×0.86 (-14%)
- いじっぱり (▲おてスピ) → ×0.9 (-10%)

```
0.75 × 0.86 × 0.9 = 0.5805
```
→ 基準4400秒のポケモンなら 4400 × 0.5805 ≒ 2554秒

## 実装

### `data/sleep_ribbon.json`
4段階のレコード。各レコードに `increment`（その段階で新たに加わる値）と `cumulative`（累計）の両方を保持。

### `utils/sleep_ribbon.py`
| 関数 | 役割 |
|---|---|
| `count_remaining_evolutions(species)` | evolution.json を再帰的に辿って残り進化回数を返す（0/1/2） |
| `get_time_multiplier(species, stage)` | 段階×種族から時間倍率を返す |
| `get_inventory_bonus(stage)` | 段階の所持数ボーナス累計 |
| `compose_total_time_multiplier(...)` | リボン×性格×サブスキルの合成倍率 |
| `stage_for_hours(hours)` | 累積眠時間から到達段階を逆引き |
