# メインスキル仕様ノート — 個体値評価チェッカー実装の参考資料

このファイルは AI（Claude等）が読み取って、`utils/evaluator.py` などで個体値評価ロジックを書くときに参照する想定の参考資料です。
効果量テーブル本体は `data/main_skill.json` に置く前提で、ここではそれだけでは見えない「評価ロジックに効く構造的な特徴」を中心にまとめます。

データソース: `貼り付けデータ集/メインスキル/*.txt`（ポケモンスリープWiki由来）。
最終更新時点: Ver.3.2.0 前後の値を採用。Ver.3.2.0以前の旧値は併記がある場合のみ参考として残します。

## 共通仕様（全スキル横断）

### フィールドボーナスの扱い
- **エナジーチャージ系のみ**、効果量にフィールドボーナスが加算される。ステータス画面ではボーナス前の値が表示される（実発動時の獲得は加算後）。
- 例: フィールドボーナス +70%、S/固定/Lv.2 = 569 → 968（小数点以下切り上げ、表示値は切り捨てで967）。

### げんき回復の上限
- げんき回復系メインスキル（オール/エール/チャージ）は **150%まで回復可能**。
- 150%超過分は切り捨て（特にげんきチャージで重要：自己回復オンリーなので過剰になりやすい）。

### ランダム版（S/ランダム）の期待値
- 効果量レンジは **min = 固定版 × 0.5、max = 固定版 × 2.0**。
- 平均（一様分布仮定）= 固定版 × 1.25（Lv7まで）。
- ゆめのかけらゲットS のみ Lv8 で 0.46〜1.84倍 → 平均 1.15倍。
- 検証データでもこの仮説は支持されている（Wiki: 検証/ランダムスキルの分布）。

### メインスキルレベル上限
- 多くのスキルは max_lv=6 か 7、ゆめのかけらゲットS のみ max_lv=8。
- 進化ボーナス＋サブスキル（スキルレベルアップM/S）だけでは Lv7 や 8 に届かないので、**メインスキルのたねが必須**。
- ヒーラー系（オール/エール）は max_lv=6。

### 派生スキルの扱い
- **派生スキルの効果量は親と同じとは限らない**（特にナイトメア＝エナジーチャージM派生は別仕様）。
- 現状確認済み: つきのひかり / つくしぐみ / たくわえる など → 親と同じ前提で仮置き。
- ナイトメア / きのみジュース / みかづきのいのり / ばけのかわ などは個別調査が必要。
- 評価ロジックでは「派生スキル名 → 親カテゴリ」のマッピングは `main_skill.json` の `category` フィールドで引ける想定。ただし数値は派生固有のテーブルを優先する。

## スキル別ノート

### エナジーチャージS / エナジーチャージM
**評価への影響度: 高**

#### 概要
直接エナジーを増やす、最も多くのポケモンが持つメインスキル系統（全マスターの21.9%）。
S/固定、S/ランダム、M/固定 の3バリアント。

#### 評価ロジックで考慮すべき点
- **レベル依存性が逆方向**: 効果量はポケモンLvで変化しない固定値。一方できのみエナジーはLvと共に増えるので、ポケモンLvが上がるほど **エナジーチャージの相対価値は下がる**。
- 評価スコアにポケモンLvの逆相関を入れる余地がある（特にスキルとくい × エナジーチャージ持ち）。
- M ≈ S × 2 倍の効果量。S/ランダムの平均は S/固定 × 1.25。
- **Lv5から効果量の伸びが大きくなる**（指数的）→ Lv4とLv6で評価重みを変えるべき。
- 「カビゴン週末・週前半」運用論: メインスキルLv高ければ日曜発動可で週明けスタートダッシュに使える。

#### 効率比較（参考、Wiki由来 Ver.3.0.0）
- ライチュウ（きのみとくい、好物前提）と各エナジーチャージ持ちスキルとくいの100秒あたりエナジー比較:
  - **Lv30以下では好物前提ライチュウより、ゴルダック・ウソッキー・デンリュウ・エーフィなど多くのスキルとくいが上**
  - **Lv50以上では好物前提のきのみとくいが逆転**
  - ゴルダック（S/ランダム、12.5%発動）はエナジーチャージ持ちの中で **好物無視前提なら全Lvで最高効率**
- 判断基準にする場合は、ポケモンLv・好物フィールドかどうか・スキル発動率を組み合わせる。

### げんきオールS
**評価への影響度: 高 / 性格依存性: 強**

#### 概要
チーム全員（自身含む）にげんき回復。ヒーラー系の中で扱いやすさNo.1。

#### 評価ロジックで考慮すべき点
- **性格補正が定量化されている**（無補正基準で1日通算）:
  - 上昇補正: +1.5〜+2.7%
  - 下降補正: -1.5〜-3.6%
  - **ずぶとい（げんき下降の最重）: -7%前後**（睡眠スコア100未満なら-6%程度に緩和）
  - げんき下降補正は最大でも-4%以下、そこまで深刻ではない。さみしがり個体なら通算プラスにも。
- **Lv5から急激に強くなる**（Lv4=11.4 → Lv5=15.0）。Lv4以下のげんきオールは大幅減点でよい。
- 1日3回以上発動できれば、**「げんきオール持ちが一切エナジーを稼がなくてもチーム全体としてプラス」**。常時編成価値の判定基準。
- 「ププリン厳選の目安」: スキル発生確率補正＋おてつだいスピード補正が合わせて2つ以上で1日3回以上発動可。

### げんきエールS
**評価への影響度: 高 / げんきオールとの差別化が必要**

#### 概要
ランダム1匹にげんき回復。1人あたり回復量はげんきオールの3〜4倍。

#### 評価ロジックで考慮すべき点
- Ver.1.5.0以降「チーム内で一番げんきが少ない個体が選ばれる確率が高い」（実測 ≈ 60%）。
- **Ver.3.0.0で大調整**: 効果量↓・スキル発動率↑ にバランス変更 → 「発動回数」の評価重みが増す。
- **自分が回復対象に選ばれない確率があり、本人のげんき切れリスク**。これにより本人のスキル発動回数が落ちる悪循環あり → げんきオールのヒーラーと併用する「ダブルヒーラー」運用がある。
- 常時編成ライン: メインスキルLv6で **4.16回/日発動** できれば、好物きのみ＋きのみの数Sのきのみとくいより優先採用価値あり。
- ただしげんきエールS持ちは1進化までしか実装されていないので、Lv上げにメインスキルのたねが最低1個必須。

### げんきチャージS
**評価への影響度: 中 / 自己回復のみ**

#### 概要
自分のみげんき回復。げんき管理しやすく、食材/きのみとくいの「睡眠交代要員」用途で活きる。

#### 評価ロジックで考慮すべき点
- **8時間半睡眠の1日減少げんき = 93%**。Lv5以上で1日3回以上発動できれば回復量が減少量を上回る → 睡眠不要運用が可能になる。
- 150%上限切り捨てがある → 連発でLvが高すぎると無駄が出るので「最大Lv ≠ 最強」の場合あり。
- 一緒に睡眠しない運用なら、睡眠EXP/リサーチEXPボーナスのサブスキルは活きないので相性悪い → サブスキル評価を運用想定で変える余地。
- 食材とくい × げんきチャージ持ち（ウツボット・バンギラス・カイリュー等）は2回進化＋睡眠不要運用が強い。

### 食材ゲットS
**評価への影響度: 高**

#### 概要
リサーチノート登録済みの全食材から3種ランダムを均等数で取得。

#### 評価ロジックで考慮すべき点
- **食材1個あたり平均基礎エナジー = 144.8**（全食材登録時、Ver.3.2.0以前）。
  - 一発動の期待エナジー（料理ボーナス無視）: 食材数 × 144.8。
  - 例: Lv7（24個）→ 約3475エナジー（エナジーチャージM Lv7=6858 のおおむね半分）。
- **料理ボーナスで逆転**: レシピボーナス最大3.34倍 + 食材数ボーナス最大1.78倍 を適用するとエナジーチャージMを超えうる。
- **食材バッグ満タンで発動失敗**。きのみは回収できるので、運用上はきのみで食材確率を上げて発動を維持する小ワザがある。
- スキルとくい食材ゲット（**シャワーズ・ヘラクロス**）は食材とくい以上の食材調達効率を出せる。
- リサーチノートに未登録の食材がある段階では効果が落ちる（特に序盤）。
- 食材セレクトS との比較: こちらは絞り込みなしの3種ランダム。狙い食材があるなら食材セレクトS が上。

### 料理パワーアップS
**評価への影響度: 中 / 特殊軸（重ねがけ可能）**

#### 概要
次回料理時のなべ容量を増やす。料理しなければ週/フィールド跨いでも維持される。重ねがけ可能、上限+200個。

#### 評価ロジックで考慮すべき点
- **単体評価より「複数体保有でローテ可能か」が運用価値に効く**特殊軸。
- いいキャンチケ補正(1.2倍)が増分にも適用される。
- 序盤は食材供給不足で活きにくい → ポケモンLv30以上＋食材ゲット持ちと併用が前提。
- 「日曜鍋パ」運用: 日曜に発動を貯めて月曜大型料理に投入する戦術がある。
- 中盤以降は最上級料理（必要食材77〜100個）への対応が必須。Lv5〜7で対応可能料理が広がる。
- なべ容量目標ライン:
  - 49: サラダ/デザートの中級
  - 55: 全カテゴリ中〜上級
  - 69: 全カテゴリ上級3種
  - 77: カレー/デザートの準最上級2種

### ゆめのかけらゲットS
**評価への影響度: 中 / 評価軸が他と異なる**

#### 概要
ゆめのかけらを直接獲得。**唯一エナジーに関与しないメインスキル**。S/固定 と S/ランダム の2バリアント、max_lv=8。

#### 評価ロジックで考慮すべき点
- **エナジー軸の評価では低くなる**。代わりに「ゆめかけら需要」軸を別途設ける必要あり。
- 評価が変動する条件:
  - **エナジーが稼ぎにくい時（低Lv帯・低フィールドボーナス）→ 価値↑**
  - **エナジーを充分稼げる時（高フィールドボーナス・育成済み）→ 価値↓**
  - **EXフィールドでは価値が約1/3**（要検証だが体感）
  - **アチーブメント完走後は供給減 → 価値↑**
- 換算ガイドライン: **12000ねむけパワー = ゆめかけら1個 ≈ 睡眠スコア100時カビゴンエナジー120**。
  - Lv8固定 = 2500個 = 30万カビゴンエナジー = 3000万ねむけパワー相当。
- ランダム版は固定版の0.5〜2.0倍レンジ、Lv7まで平均1.25倍、Lv8で1.15倍。
- 持つポケモンの多くがスキルとくいでエナジー稼ぎが弱い → 編成枠コストが他より高い。
- 1進化/無進化のみなのでLv上げにたね必須。
- 全ポケモン中マルノームが最も多くのゆめかけらを稼げる（参考、Ver.1.8.0時点）。

## 評価ロジック実装時の参考方針（提案）

このノートを踏まえた評価ロジックの組み立て方の叩き台:

1. **スキルカテゴリごとに評価ウェイト分岐**: エナジー直結（エナチャ/食材ゲット）と、エナジー間接（オール/エール/チャージ）と、エナジー外（ゆめかけら）で別軸。
2. **ポケモンLvとスキルLvの組み合わせで補正**: 特にエナジーチャージは「Lv30以下で高評価」のような区間補正を入れる。
3. **性格補正の重み**: げんきオール持ちはずぶとい-7%、げんき下降-3%等を効果量に直接適用。
4. **発動回数閾値**: ヒーラー系は「3回/日」「4.16回/日」がライン。スキル発動率×おてつだい時間×補正で算出。
5. **派生スキルの未調査リスク**: ナイトメア・きのみジュース・みかづきのいのり等は親値で代用しつつ、評価結果に「※派生未調査」フラグを残す。
6. **ランダム版の評価**: 期待値（固定×1.25）で評価しつつ、ぶれの大きさをユーザー設定で重み調整できるようにする余地。
